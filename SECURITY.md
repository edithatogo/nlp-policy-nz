# Security Policy

## Supported Versions

Security fixes are applied to the active main branch and tagged releases when a
relevant issue is identified.

## Reporting a Vulnerability

Do not open a public issue for a suspected vulnerability.

Use one of the following private reporting channels instead:

- GitHub Security Advisories for this repository
- A private message to the repository maintainers if an advisory is not
  available

Include:

- A short summary of the issue
- Affected file or feature paths
- Reproduction steps
- Any mitigation you know about

## Response Expectations

The maintainers will acknowledge the report, assess severity, and coordinate a
fix or follow-up disclosure path as appropriate.
